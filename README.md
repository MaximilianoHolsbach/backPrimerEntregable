# backPrimerEntregable
Primer entregable para la clase de back end de Coder House

```markdown
# DESAFÍO ENTREGABLE

## Clases con ECMAScript y ECMAScript avanzado

### Consigna:

- Programar una clase "ProductManager" que gestione un conjunto de productos.
- Programar una clase "UserManager" que gestione un conjunto de usuarios.

### Aspectos a incluir:

- Debe crearse con una variable privada y que guarde todos los productos/usuarios en un arreglo.

#### Propiedades de cada producto:

- id (código identificador)
- title (título)
- photo (ruta de imagen)
- price (precio)
- stock (unidades disponibles)

#### Propiedades de cada usuario:

- id (código identificador)
- name (nombre)
- photo (ruta de imagen)
- email (correo electrónico)

#### Métodos de cada clase:

- `create(data)`: Agrega un producto/usuario al arreglo de productos/usuarios inicial. Todos los campos son obligatorios, excepto id que se agrega automáticamente y es autoincrementable.
- `read()`: Devuelve el arreglo con todos los productos/usuarios.
- `readOne(id)`: Devuelve el objeto producto/usuario buscado.

### Formato del entregable:

- Pull Request (PR) de rama sprint1 hacia main/master según corresponda.
- Prueba de los métodos y capturas de pantalla incluidas en la PR.
- Incluir readme.md explicando lo que se entregó.
- La entrega es individual. En caso de trabajar en parejas, informar al tutor con quién trabajaron (ambos deben avisar).
```
